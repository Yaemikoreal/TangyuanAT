# 自主任务分配规则

## 额度管理原则

1. **每日额度预算**：有限，优先关键任务
2. **检查频率**：每 4 小时一次（08:00, 16:00）
3. **任务触发条件**：仅当实际需要时创建任务
4. **避免浪费**：不重复检查已完成任务

---

## 任务优先级矩阵

| 优先级 | 条件 | 处理策略 |
|--------|------|----------|
| 🔴 P0 | Bug 影响核心功能 | 立即处理，占用额度 |
| 🟡 P1 | 新功能开发 | 正常工作时间内处理 |
| 🟢 P2 | 优化改进 | 额度充足时处理 |
| ⚪ P3 | 文档/美化 | 低优先级，延后处理 |

---

## 自动任务分配逻辑

### 检查触发条件

```
IF GitHub Issues 有 open 状态:
    IF Issue 标签为 "bug" OR "critical":
        立即分配给昔涟处理
    ELSE IF Issue 标签为 "feature":
        评估工作量，分配给汤圆或豆腐
    ELSE:
        加入待处理队列
ELSE:
    跳过检查，节省额度
```

### Agent 负载均衡

| Agent | 专长 | 负载上限 |
|-------|------|----------|
| 昔涟 | 架构/复杂问题 | 2 任务/天 |
| 汤圆 | 开发/前端 | 3 任务/天 |
| 豆腐 | 测试/运维 | 2 任务/天 |
