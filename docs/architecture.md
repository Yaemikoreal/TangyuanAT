### 核心模块说明

#### 1. 数据模型层 (models/)

**文件**: `models/database.py`

**职责**:
- 定义数据库表结构
- 提供 ORM 映射
- 数据库初始化

**核心模型**:

| 模型 | 表名 | 说明 |
|------|------|------|
| `Agent` | agents | Agent 状态信息 |
| `WorkLog` | work_logs | 工作日志 |
| `ChatHistory` | chat_history | 对话历史 |
| `StatRecord` | stats | 统计记录 |
| `Config` | config | 系统配置 |
| `Alert` | alerts | 告警记录 |
| `AlertRule` | alert_rules | 告警规则 |
| `AlertNotification` | alert_notifications | 告警通知 |

#### 2. Agent 模块 (agents/)

**文件**: `agents/openclaw_client.py`

**职责**:
- 与 OpenClaw Gateway 通信
- 获取 Agent 实时状态
- 会话信息查询

**核心功能**:

```python
class OpenClawClient:
    def get_status()           # 获取网关状态
    def get_sessions()         # 获取会话列表
    def get_agent_status()     # 获取 Agent 状态
    def get_agent_history()    # 获取历史消息
```

#### 3. 告警模块 (Alert Manager)

**集成在**: `app.py`

**职责**:
- 监控 Agent 状态
- 触发告警规则
- 发送通知

**告警类型**:

| 类型 | 触发条件 | 严重级别 |
|------|----------|----------|
| `offline` | Agent 离线超过阈值 | warning |
| `task_failed` | 任务执行失败 | error |
| `response_slow` | 响应时间过长 | warning |
| `custom` | 自定义告警 | 可配置 |

#### 4. 监控模块 (Monitoring)

**集成在**: `app.py`

**职责**:
- 系统资源监控
- 数据库健康检查
- 网关状态检查

**监控指标**:

- CPU 使用率
- 内存使用率
- 磁盘使用率
- 数据库连接状态
- 网关连接状态

### 数据表详解

#### Agent 表

```sql
CREATE TABLE agents (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    english_name VARCHAR(100),
    status VARCHAR(20) DEFAULT 'offline',
    model VARCHAR(50),
    role VARCHAR(100),
    character TEXT,
    emoji VARCHAR(10),
    last_active DATETIME,
    tasks_completed INTEGER DEFAULT 0,
    messages_processed INTEGER DEFAULT 0,
    current_task TEXT,
    skills TEXT,  -- JSON array
    config TEXT,  -- JSON object
    created_at DATETIME,
    updated_at DATETIME
);
```

#### WorkLog 表

```sql
CREATE TABLE work_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_id VARCHAR(50) NOT NULL,
    agent_name VARCHAR(100),
    action VARCHAR(200) NOT NULL,
    details TEXT,
    timestamp DATETIME,
    FOREIGN KEY (agent_id) REFERENCES agents(id)
);
```

## 接口设计

### RESTful API 设计原则

- 使用名词表示资源
- HTTP 方法表示操作
- 统一响应格式

### 响应格式

**成功响应**:

```json
{
  "success": true,
  "data": { ... },
  "timestamp": "2026-03-15T12:00:00"
}
```

**错误响应**:

```json
{
  "success": false,
  "error": "错误描述"
}
```

### API 端点设计

| 资源 | 端点 | 方法 | 操作 |
|------|------|------|------|
| **Agent** | `/api/agents` | GET | 列表 |
| | `/api/agents/{id}` | GET | 详情 |
| | `/api/agents/{id}` | PUT | 更新 |
| **日志** | `/api/logs` | GET | 列表 |
| | `/api/logs` | POST | 创建 |
| **告警** | `/api/alerts` | GET | 列表 |
| | `/api/alerts/{id}` | GET | 详情 |
| | `/api/alerts/{id}/acknowledge` | POST | 确认 |
| | `/api/alerts/{id}/resolve` | POST | 解决 |
